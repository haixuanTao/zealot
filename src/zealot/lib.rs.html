1//! # zealot — GPU-resident RL for the LeRobot bipedal
2//!
3//! All-Rust PPO training for the LeRobot bipedal on dimforge's GPU physics
4//! stack (nexus rigid-body sim + vortx tensors + khal backends). The whole
5//! loop — physics, policy forward, PPO update — runs on the GPU; this crate
6//! hosts the runnable examples under `examples/` (there is no library API).
7//!
8//! ## Pick your platform
9//!
10//! | you have | guide | backend |
11//! |---|---|---|
12//! | an NVIDIA box (RTX 5090 / sm_120) | [`guides::train_on_5090`] | native CUDA (cuda-oxide) |
13//! | a Mac (Apple Silicon) | [`guides::train_on_macos`] | WebGPU → Metal |
14//!
15//! Both guides produce the same trainer, checkpoints, and (verified) the
16//! same physics to ~1e-4 — pick by hardware, not by feature set.
17//!
18//! ## Quick start (either platform)
19//!
20//! ```bash
21//! #                                        iters  num_envs  checkpoint
22//! cargo run --release --example biped_train_gpu \
23//!     --features "gpu biped_gpu" --        2000   1024      walking_policy.safetensors
24//! ```
25//!
26//! The backend auto-selects (CUDA on sm_120, otherwise WebGPU). See your
27//! platform guide for the required sibling checkouts and one-time setup —
28//! **macOS needs the vendored-naga patch clone** or the sim silently breaks.
29
